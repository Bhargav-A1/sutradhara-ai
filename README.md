Component Documentation: Localized Intelligence Engine and Secure Identity Gateway
1. High-Level Executive Summary
This component serves as the localized intelligence core and secure identity gateway for the Sutradhara AI platform. Developed during an intensive single-day development sprint, the module bridges external national data registries, specifically Digital India's API Setu and DigiLocker systems, with an autonomous, open-source compliance engine running entirely on local host architecture.

The primary objective of this module is to act as a secure validation layer. It ingests user verification credentials, extracts critical demographic insights, and computes a dynamic, data-driven Expat Stability Score to inform and guide downstream cross-border financial automation workflows.

2. Core Technical Architecture and Implementation
2.1 Standardized API Architecture and Route Mapping
Objective: Map the cross-border authentication pipeline to allow users to securely connect their verified identification assets, such as Aadhaar or Permanent Account Number (PAN) registries, to the platform.

Implementation: A standardized OAuth 2.0 Authorization Code Flow specification was architected and documented. To ensure immediate developer onboarding, fallback testing loops were configured to route through local browser callback interceptors located at localhost:8080/callback. This established a decoupled data sandbox environment, enabling rapid prototyping without the typical friction or delays associated with organizational KYC approvals.

2.2 Low-Footprint Local LLM Deployment
Objective: Parse raw, unstructured user profiles into structured compliance tokens efficiently, eliminating dependency on costly cloud APIs while keeping sensitive user data entirely isolated within the local system.

Implementation: The architecture utilizes Alibaba’s Qwen 1.5B Instruct model, deployed via the high-performance GGUF binary standard. By leveraging the llama-cpp-python library, the model compiles directly into the system's native CPU matrix layout. This approach ensures highly efficient and rapid token inference within standard system RAM limits, completely bypassing the driver conflicts and memory allocation errors often encountered with graphics hardware.

2.3 Deterministic Extraction and Scoring Pipeline
Objective: Convert natural language or irregular model outputs into strict, predictable data types required by automated state managers and orchestration layers like LangGraph.

Implementation: * Implemented a cold-temperature, ChatML-formatted system prompt that restricts the model’s operations to that of a rigid data parser.

Integration of a robust regular expression (Regex) pattern matcher designed to isolate the outer curly braces of the output string. This step successfully strips away any conversational prefixes or markdown formatting artifacts generated by the model.

Engineered an algorithmic scoring matrix that dynamically weighs the extracted verification fields and structural risk markers to output a real-time risk index normalized between 0 and 100.
